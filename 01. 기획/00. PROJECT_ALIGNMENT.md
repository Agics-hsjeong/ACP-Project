# 3. Dynamic Interaction Loop

Living World Engine의 핵심 루프. 모든 기획·UI는 이 순환을 따른다.

```text
① 대화 & 상호작용 (RP Chat)
        ↓
② 에피소드 기억 생성 (Memory Engine)
        ↓
③ 감정 · 관계망 변동 (Emotion · Relationship Engine)
        ↓
④ 캐릭터 행동 모델 변화 (Prompt System)
        ↓
    (①으로 반복)
```

**전환 비용(Switching Cost) 전략:** 루프 반복 시 기억·감정·관계 데이터가 누적되어 이탈 비용이 증가한다.

# 5. Emotion System Standard

**공식 감정 6축 (전 문서 통일)**

| Key | 한글 | 범위 |
|-----|------|------|
| affection | 애정 | 0~100 |
| trust | 신뢰 | 0~100 |
| respect | 존경 | 0~100 |
| anger | 분노 | 0~100 |
| fear | 공포 | 0~100 |
| jealousy | 질투 | 0~100 |

> **주의:** 초기 PDF/UI 초안의 "불안(Anxiety)"는 **공포(fear)** 로 통일한다.  
> RP Chat 실시간 미니 차트는 affection · trust · anger · fear 4축 요약 표시 가능 (Phase 1).

# 7. Performance Targets

| 지표 | Phase 1 목표 | 비고 |
|------|-------------|------|
| First Token (SSE) | < 2s | RAG + Prompt 조립 포함 |
| Full Response | < 8s | 스트리밍 완료 기준 |
| Memory Retrieval | < 500ms | pgvector Top-K |
| API Availability | 99.5% | PRODUCT_SPEC |

> 기존 "평균 응답 < 1s" 목표는 RAG + Gemini + Validator 파이프라인과 불일치하여 수정함.
