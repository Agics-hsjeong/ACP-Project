# 1. Product Overview

## 1.1 Product Name

AI Character Playground (ACP)

## 1.2 Product Description

AI Character Playground는 창작자가 세계관과 캐릭터를 제작하고, 사용자는 해당 캐릭터와 장기 기억(Long-Term Memory)을 기반으로 역할극(Roleplay)을 수행할 수 있는 AI 스토리 플랫폼이다.

본 문서는 서비스의 기능 명세, 화면 정의, 사용자 흐름 및 비기능 요구사항을 정의한다.

# 5. Functional Requirements

### Description

사용자는 캐릭터를 검색할 수 있다.

### Input

- Keyword
- Tag
- Genre

### Output

- Character List

---

## F002 Session Creation

### Description

사용자는 새로운 대화 세션을 생성할 수 있다.

### Flow

1. Character Selection
2. Session Creation
3. Chat Start

---

## F003 Chat Messaging

### Description

사용자는 메시지를 전송할 수 있다.

### Input

- Text Message

### Output

- Character Response

---

## F004 Memory Creation

### Description

시스템은 중요 사건을 장기 기억으로 저장한다.

### Trigger

- Significant Event
- Emotional Change
- Session Milestone

---

## F005 Emotion Update

### Description

대화 결과에 따라 감정을 갱신한다.

### Example

| Event | Effect |
|---------|---------|
| Gift | Affection +5 |
| Lie | Trust -10 |
| Betrayal | Anger +20 |

---

## F006 Relationship Update

### Description

관계 상태를 갱신한다.

### Example

Friend → Close Friend → Lover

# 6. Non Functional Requirements

## Performance

### Target (Phase 1)

- First Token (SSE): < 2s
- Full Response (Streaming Complete): < 8s
- Memory Retrieval: < 500ms
- API Availability: 99.5%

> RAG + Prompt Assembly + Gemini 파이프라인 기준.  
> 기존 "평균 < 1s" 목표는 현실과 불일치하여 수정 (`00. PROJECT_ALIGNMENT.md` §7).

---

## Availability

### Target

99.5%

## Security

### Requirements

- JWT Authentication

- Rate Limiting
- Prompt Injection Protection
