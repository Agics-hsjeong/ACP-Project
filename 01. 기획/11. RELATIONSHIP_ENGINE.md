# 1. Purpose

Relationship Engine은 사용자와 캐릭터, 캐릭터 간 관계를 관리하고 시각화한다.

목표:

- 관계의 지속적 성장
- 감정 수치와 연동
- 세계관 내 사회망 표현
- (Phase 5) 사용자 선택의 NPC 간 연쇄 영향

# 3. Relationship Stages (User ↔ Character)

```text
STRANGER → ACQUAINTANCE → FRIEND → CLOSE_FRIEND → PARTNER → LOVER
```

## 3.1 Transition Conditions (Phase 3)

| From → To | 조건 예시 |
|-----------|----------|
| STRANGER → ACQUAINTANCE | 대화 10턴+ |
| ACQUAINTANCE → FRIEND | trust ≥ 50, affection ≥ 40 |
| FRIEND → CLOSE_FRIEND | trust ≥ 70, 중요 기억 3개+ |
| CLOSE_FRIEND → PARTNER | affection ≥ 80, 고백 기억 존재 |
| PARTNER → LOVER | affection ≥ 90, trust ≥ 85 |

## 3.2 역행 (Regression)

- trust 급락 (-20 이상 단일 이벤트) 시 Stage 하향 가능
- LOVER → CLOSE_FRIEND (이별·배신 기억)

# 6. NPC Cascade System (Phase 5)

> Living World Engine PDF: "나의 선택이 NPC들 간의 관계에 연쇄적 영향"

## 6.1 Scope

사용자 행동이 **동일 세계관 내 NPC-NPC 관계**에 간접 영향을 미친다.

## 6.2 Example

```text
사용자: 카이온과 대립하여 엘리아를 도움
  ↓
엘리아 ↔ 카이온: ENEMY 강화
엘리아 ↔ 세레나: ALLIANCE 유지
루카스 ↔ 카이온: ENEMY 전파 (동맹 연쇄)
  ↓
관계 히스토리에 "전쟁의 서막" 이벤트 기록
```

## 6.3 Rules

- Cascade는 **같은 World** 내에서만 발생
- Creator가 정의한 Faction Alliance 테이블 참조
- Celery 비동기 처리 (Chat 응답 후)
- Phase 1~4: **미구현** (Lore 고정 관계만 표시)
